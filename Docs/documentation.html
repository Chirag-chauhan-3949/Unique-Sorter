Unique Sorter CRM

A complete guide to using the CRM platform built for Unique Sorter & Equipment Pvt. Ltd. — from field engineer enquiry submissions to generating professional quotations.

Overview

What this platform does and who it is for.

The Unique Sorter CRM is a web-based platform that helps the team at Unique Sorter And Equipment Pvt. Ltd. manage their entire sales workflow in one place. Field engineers on site collect customer requirements and enter them into the system. The back-office team then reviews enquiries, generates quotations, and shares professional PDFs with clients.

Enquiry Management
Field engineers capture customer requirements on site through a structured form. Track each enquiry with full details, items, pricing, and follow-up dates.
Quotation Generation
Convert enquiries into professional, GST-compliant PDF quotations with one click. Share via WhatsApp or email instantly.
Team Collaboration
All team members can view enquiries, edit details, and generate quotations. Admins get additional settings control.
Secure Access
Phone-based OTP login ensures only verified team members can access the dashboard. No passwords to remember.

Submitting an Enquiry

For field engineers — how to enter a customer's requirement after a site visit.

After visiting a client, open the CRM and navigate to the enquiry form. You can access it from the "+ New Enquiry" button on the dashboard, or directly via the /enquiry page (no login required for quick entry).

What the form asks for

Section 1 — Customer Information

Customer Name (required)
Mill / Company Name (required)
Mobile Number (required)
Email Address (optional)
GST Number (optional)
Lead Source (dropdown)

Section 2 — Location Details

City / Location (required)
State (required)
Full Address (required)

Section 3 — Requirement

You will be asked: "Do you have an immediate requirement?"

✅
If YES (Immediate): You can add one or more sorting machine items with Model (Pinnacle / Nandak), Size (5–10 chutes), Quantity, and Price. Prices are auto-calculated based on your state and chosen model.
🕑
If NO (Future): You can specify the commodity type (Rice, Pulses, etc.), expected month of order, chances of order (%), and any notes for follow-up.

Section 4 — Remarks (optional additional notes)

After filling everything, click "Save". You will see a success confirmation. The enquiry is now stored in the system and the back-office team can review it, generate quotations, and follow up.

Registration

How to create your team account.

1
Go to the Registration Page
Click "Register here" from the login page, or navigate directly to /register.
2
Fill in Your Details
Enter your full name, phone number (10 digits, starting with 6–9), choose your role (User or Admin), and set a password. You can toggle password visibility with the eye icon.
3
Submit & Login
After successful registration, you'll be redirected to the login page where you can sign in with your phone number and OTP.
⚠️
Note: Admin registration requires approval from an existing admin. New users should select the "User" role. Contact your administrator if you need admin access.

Login (Phone OTP)

Secure, passwordless login with one-time codes sent to your phone.

1
Enter Your Phone Number
On the login page, type your 10-digit phone number (the same one used during registration). The country code +91 is added automatically.
2
Receive OTP via SMS
Click "Send OTP". An invisible security check runs in the background (reCAPTCHA), and a 6-digit code is sent to your phone via SMS.
3
Enter the 6-Digit Code
Type the code into the 6 boxes. Each box auto-advances to the next. You can also paste the full code. If the code expires, click "Resend OTP" after the 60-second timer.
4
You're In!
After verification, you are automatically logged in and redirected to the dashboard. Your session stays active and refreshes automatically.

Managing Enquiries

How to view, edit, filter, and act on customer enquiries.

Viewing Enquiries

Navigate to Enquiry in the sidebar. You'll see a table listing all enquiries with columns:

Customer Name
Requirement (Immediate / Future)
Mill / Company
Mobile Number
Location
Date

While loading, animated skeleton placeholders appear instead of a spinner, so you can see the table structure taking shape.

Searching & Filtering

Use the search bar and filter panel at the top to narrow down enquiries by customer name, location, requirement type, or date range.

Viewing Details

Click any row to open the full enquiry detail page. Here you can see all the information the field engineer recorded, including items, pricing, and notes.

Editing an Enquiry

On the detail page, click the three-dot menu (top-right) and select "Edit". Make your changes and click "Save". All edits are tracked.

Creating a New Enquiry

Click the "+ New Enquiry" button at the top of the enquiry list. This opens the enquiry form where you can enter the customer's details collected during a site visit.

Deleting an Enquiry

From the three-dot menu on the detail page, select "Delete". You'll be asked to confirm before the enquiry is permanently removed.

Admin Settings

Platform configuration available to administrators only.

The Settings page is accessible only to users with the Admin role. It appears in the sidebar navigation only for admins.

From Settings, admins can manage platform-level configuration such as Firebase database connections and system preferences.

⚠️
If you see an "Access Denied" message when trying to open Settings, your account has the User role. Ask an administrator to upgrade your access if needed.

Per-User Screen Access

⚠️ Planned Feature — Not Yet Implemented

Allow admins to control exactly which dashboard screens each user can access.

What It Will Do

Currently, all users (with the "User" role) see the same sidebar: Dashboard, Enquiry, and Quotations. With per-user screen access, an admin will be able to grant or revoke access to specific screens for each individual user.

For example, a field engineer might only need access to the Enquiry page, while a sales manager might need Enquiry + Quotations + Reports.

How It Should Work (User Experience)

1
Admin Opens Settings → User Management
The admin navigates to the existing Settings page and clicks "Edit" on any user.
2
Screen Access Checkboxes Appear
Below the Role selector in the Edit User modal, a new "Screen Access" section shows a checkbox for each dashboard screen: Dashboard, Enquiry, Quotations, Profile, Reports, Products, Orders, Inventory, Clients.
3
Admin Toggles Screens On/Off
The admin checks or unchecks screens. "Dashboard" and "Profile" should always remain on (non-removable). Save writes to Firestore.
4
User Sees Only Allowed Screens
When the user logs in, the sidebar shows only the screens the admin has allowed. Navigating to a blocked URL redirects to the dashboard.

Implementation Guide

1. Firestore Data Model

Add an allowedScreens array to each user document in the userdata collection:

// Firestore: userdata/{phone} { "name": "Ashok Sharma", "phone": "9876543210", "role": "USER", "allowedScreens": ["dashboard", "enquiry", "quotations", "profile"], "createdAt": "2026-01-15T..." }

Admins always have access to all screens regardless of this field.

2. Backend — Return allowedScreens on Login

In src/app/api/auth/verify-firebase-token/route.js, include the field in the response:

// Inside the POST handler, after fetching userDoc.data() return NextResponse.json({ success: true, user: { id: userId, name: data.name, phone, role, allowedScreens: data.allowedScreens || null, }, });

3. Frontend — AuthContext

In src/contexts/AuthContext.js, store allowedScreens alongside user data. On the route guard, if the user has an allowedScreens array and the current page's key is not in it, redirect to /dashboard.

// In the route guard useEffect const screens = user?.allowedScreens; if (screens && userRole !== 'ADMIN') { const pageKey = pathname.split('/')[2] || 'dashboard'; if (!screens.includes(pageKey)) { router.push('/dashboard'); return; } }

4. Frontend — Sidebar Filtering

In src/components/Sidebar.js, filter the NAV array using allowedScreens:

// Add a "key" to each NAV item: e.g. { key: 'enquiry', href: '/dashboard/enquiry', ... } const filteredNav = NAV.filter(item => { if (!item.allowedRoles.includes(userRole)) return false; if (userRole === 'ADMIN') return true; const screens = user?.allowedScreens; if (!screens) return true; // no restriction set = show all return screens.includes(item.key); });

5. Settings Page — Screen Access Editor

In the UserModal component within src/app/dashboard/settings/page.js, add a checkbox group below the Role toggle. Each checkbox maps to a screen key. On save, write allowedScreens to the user's Firestore document.

💡
Default behaviour: If allowedScreens is not set (null/undefined) for a user, they should see all screens their role allows. This maintains backward compatibility with existing users.

Implementation Guide

Approach: Client-side CSV Generation

Since the enquiry page already loads all enquiries into the rows state array, CSV generation can be done entirely in the browser — no new API endpoint needed.

1. Add the export function to src/app/dashboard/enquiry/page.js:

function downloadCSV(rows) { const headers = [ 'Customer Name', 'Mill/Company', 'Mobile', 'Email', 'GST No.', 'City', 'State', 'Address', 'Has Requirement', 'Lead Source', 'Items', 'Commodity', 'Remarks', 'Date' ]; const escape = v => { const s = String(v ?? ''); return s.includes(',') || s.includes('"') || s.includes('\n') ? '"' + s.replace(/"/g, '""') + '"' : s; }; const csvRows = rows.map(r => { const items = (r.items || []) .map(i => `${i.model} ${i.size}ch x${i.qty}`) .join(' | '); return [ r.customerName, r.millName, r.mobile, r.email, r.gstNo, r.location, r.state, r.address, r.hasRequirement ? 'Yes' : 'No', r.leadSource, items, r.commodity, r.remarks, r.createdAt ].map(escape).join(','); }); const csv = [headers.join(','), ...csvRows].join('\n'); const blob = new Blob([csv], { type: 'text/csv' }); const url = URL.createObjectURL(blob); const a = document.createElement('a'); a.href = url; a.download = `enquiries-${new Date().toISOString().slice(0,10)}.csv`; a.click(); URL.revokeObjectURL(url); }

2. Add the button in the topbar area of the enquiry page (next to "+ New Enquiry"):

{/* Only show to admins */} {isAdmin(userRole) && ( <button onClick={() => downloadCSV(rows)}> Download CSV </button> )}

Data & Storage

Where your data lives and how it is protected.

Firebase Firestore
All data (user accounts, enquiries, quotations) is stored in Google's Firebase Firestore — a secure, cloud-hosted database.
Firebase Authentication
Login is handled by Firebase Auth with phone-based OTP. No passwords are stored in the database. Sessions are encrypted with JWT tokens.

Frequently Asked Questions

Common questions and quick answers.

I'm not receiving the OTP. What should I do?

Wait for the full 60-second timer to expire, then click "Resend OTP". Ensure your phone number is correct and has SMS reception. If the issue persists, check if you have registered with that phone number first.

Can customers see the dashboard?

No. The dashboard is only accessible to registered and logged-in team members (field engineers, back-office staff, admins). Customers never interact with the platform directly.

Can I edit a quotation after saving it?

Yes. Open the quotation from the quotations list and modify any field. You can re-download or re-share the updated PDF.

How are prices calculated?

Prices are auto-filled based on a lookup table that uses the customer's state, machine model, and size (chutes). These can be manually overridden in the quotation editor if needed.

Who can see my data?

All logged-in team members (both Users and Admins) can see all enquiries and quotations. There is no per-user data isolation — it's a shared team workspace.

Is the platform mobile-friendly?

Yes. Every page is fully responsive. On phones, the sidebar becomes a slide-out menu, tables adapt to show key info in a compact layout, and all forms are touch-friendly.

What browsers are supported?

Any modern browser — Chrome, Firefox, Safari, Edge. We recommend keeping your browser updated for the best experience.
