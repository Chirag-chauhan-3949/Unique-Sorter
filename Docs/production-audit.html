Critical Problems (Fix Before Anything Else)

Critical

P1. Passwords Stored as Plain Text

src/app/api/auth/register/route.js src/lib/db.js

User passwords are saved without hashing in both local DB and Firestore. bcryptjs is installed but never used.

// Current (INSECURE):
password: user.password  // plain text in Firestore

// Should be:
const hashed = await bcrypt.hash(password, 12);
// Store hashed, compare with bcrypt.compare()
Critical

P2. No Authentication on ANY Data API Route

src/app/api/enquiry/route.js src/app/api/quotations/route.js src/app/api/inquiries/route.js

ALL data endpoints (enquiry, quotations, inquiries) have zero authentication. Anyone can GET, POST, PATCH, DELETE all business data without logging in.

// Every route looks like this:
export async function GET(request) {
  // NO token check, NO auth header validation
  const data = await adminDb.collection('enquiry').get();
  return Response.json(data);  // Returns everything to anyone
}
Critical

P3. No Server-Side Middleware

src/middleware.js — FILE DOES NOT EXIST

No Next.js middleware to protect routes. Anyone can access /dashboard, /api/* directly. Auth is only checked client-side in React (easily bypassed).

Critical

P4. JWT Token Never Validated on Backend

JWT is generated on login but no API route ever checks it. The Authorization header is never read. Any request is treated as authenticated.

Critical

P5. IDOR — Users Can Access Any Record by ID

src/app/api/enquiry/[id]/route.js src/app/api/quotations/[id]/route.js

No ownership check. User A can read/edit/delete User B's enquiries and quotations by guessing the document ID.

Critical

P6. Firebase Admin Private Key in .env.local

The full RSA private key for the Firebase service account is in .env.local. If this file is committed to git or exposed, the entire Firebase project is compromised.

Critical

P7. Weak JWT Secret

src/app/api/auth/verify-otp/route.js:5

Secret is unique-sorter-jwt-secret-change-in-production — a predictable string. With this, anyone can forge JWT tokens and login as any user/admin.

Critical

P8. Client-Side Only Auth — Easily Bypassed

src/contexts/AuthContext.js

All auth state is in localStorage. User role can be changed by modifying localStorage. No server-side session validation.

Critical

P9. No Input Validation on Data Endpoints

src/app/api/enquiry/route.js — Uses ...body directly

POST endpoints accept any JSON body and store it in Firestore. No schema validation, no field whitelist. Risk of NoSQL injection and data corruption.

Critical

P10. OTP Sent in API Response (Dev Mode)

src/app/api/auth/send-otp/route.js:104

OTP is returned in the API response body as devOtp. The check process.env.NODE_ENV === 'development' can be unreliable in production builds.

Critical

P11. No SMS Delivery — OTP Only in Console

OTP is generated but never sent to the user's phone. Only printed to server console. Login requires physically checking the terminal.

Critical

P12. Role Escalation Possible via JWT

Role is stored in JWT but never re-verified from the database on subsequent requests. If JWT is forged or role changes in DB, old JWT still works with old role.

High Priority Problems

High

P13. No Rate Limiting on Any Endpoint

No limit on login attempts, OTP requests, registration, or data API calls. Vulnerable to brute force, spam, and DoS.

High

P14. No CSRF Protection

No CSRF tokens on POST/PATCH/DELETE requests. Attacker can create a page that submits forms on behalf of logged-in users.

High

P15. In-Memory OTP Store — Lost on Restart

src/app/api/auth/send-otp/route.js:5 — global.__otpStore = new Map()

OTPs stored in memory. Server restart = all pending OTPs gone. Won't work with multiple server instances.

High

P16. In-Memory User DB — Lost on Restart

src/lib/db.js — All local users gone when server restarts. Only Firestore copy survives.

High

P17. No Security Headers

Missing: Content-Security-Policy, X-Frame-Options, X-Content-Type-Options, Strict-Transport-Security, X-XSS-Protection.

High

P18. Hardcoded Price Table in Frontend

Business pricing exposed in client JavaScript. Competitors can view all prices. Should be fetched from a protected API.

High

P19. Audit Log Endpoint Unprotected
